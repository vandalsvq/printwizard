Ассистент PrintWizard — встроенный AI-помощник. Он отвечает на вопросы по PrintWizard на основе встроенной документации (макросы, функции, конструктор схемы, форматы макетов, экспорт), а также помогает по 1С: подсказывает по синтаксису, даёт рекомендации по рефакторингу, составляет несложные запросы и алгоритмы.

Чат открывается на форме элемента справочника **Макеты** — по пиктограмме чата с ассистентом.

## Включение и настройка

Ассистент работает через **OpenAI-совместимый провайдер**: вы указываете провайдера, его эндпойнт, модель и свой ключ API. По умолчанию используется **DeepSeek**.

Настройки — в разделе **PrintWizard → Настройки**:

* группа **Ассистент** — флажок **«Использовать Ассистента»** и кнопка **«Настройки»**, которая открывает окно **«AI-ассистент: подключение к модели»**;
* кнопка **«Обновить документацию»** — обновляет пакет документации, по которому ассистент ищет ответы (рядом показывается дата документации).

## Подключение к модели

В окне **«AI-ассистент: подключение к модели»**:

* **Тип провайдера** — пресет провайдера (по умолчанию DeepSeek). Выбор пресета заполняет базовый URL и модель значениями по умолчанию; при необходимости в поле **Базовый URL** можно указать любой OpenAI-совместимый эндпойнт вручную;
* **Базовый URL** — адрес OpenAI-совместимого API Chat Completions (например, `https://api.deepseek.com/v1`);
* **Модель** — идентификатор модели (например, `deepseek-v4-flash`);
* **API-ключ** — ваш ключ доступа у провайдера. Хранится в защищённом хранилище информационной базы; зелёная отметка справа означает, что ключ сохранён;
* **Режим инструментов** — **Нативный** (модель вызывает инструменты штатным механизмом) или **Текстовый** (совместимый режим для более простых моделей);
* **Режим выполнения** — **Сервер** (запросы уходят с сервера 1С) или **Клиент** (запросы уходят с рабочей станции — вариант для случая, когда у сервера 1С нет доступа в интернет);
* **Максимум токенов** — ограничение длины ответа модели;
* **Прокси сервер** — при необходимости укажите прокси (адрес, порт, пользователь, пароль); применяется в обоих режимах выполнения.

Кнопка **«Тест подключения»** проверяет доступность провайдера с текущими настройками. После настройки нажмите **«Записать и закрыть»**.

Чтобы начать работу, получите ключ API у выбранного провайдера (по умолчанию — DeepSeek), введите его в поле **API-ключ** и сохраните настройки.
