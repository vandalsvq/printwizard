# Как редактировать документацию (Astro Starlight)

Памятка по оформлению статей сайта [printwizard.ru](https://printwizard.ru/).
Документация живёт в **`src/content/docs/`** — это единственный источник. Правишь `.md` там, открываешь PR, после merge в `master` сайт деплоится автоматически.

> Не редактируй `/docs-llm/` (генерируется автоматически) и не создавай `/docs` (его больше нет).

---

## 1. Где лежит статья

Адрес на сайте ↔ файл:

| Адрес | Файл |
|---|---|
| `/guide/ch-01-01/` | `src/content/docs/guide/ch-01-01.md` |
| `/guide/` (обложка раздела) | `src/content/docs/guide.md` |
| `/model/ch-01-02/` | `src/content/docs/model/ch-01-02.md` |

Имя файла = slug в URL. Используй латиницу, нижний регистр, дефисы: `моя-статья.md` → лучше `my-article.md`.

---

## 2. Frontmatter (шапка статьи)

Каждый файл начинается с блока между `---`. Обязателен только `title`.

```md
---
title: "Заголовок статьи"      # обязательно — это H1 страницы
description: "Краткое описание для поиска и превью"   # опционально
---

Текст статьи…
```

Полезные поля (все опциональны):

```md
---
title: "Заголовок"
description: "…"
tableOfContents: false          # убрать оглавление справа
slug: "custom/url"              # переопределить URL (обычно не нужно)
draft: true                     # черновик — не попадёт в прод-сборку
pagefind: false                 # исключить из поиска
---
```

> `sidebar:`-поля (`order`, `label`, `hidden`) во frontmatter у нас **не управляют меню** — меню ручное (см. §3).

---

## 3. Меню (навигация) — `src/sidebar.mjs`

Меню слева задаётся **вручную** в [`src/sidebar.mjs`](src/sidebar.mjs). Порядок пунктов = порядок в массиве.

Форматы пунктов:

```js
'guide/ch-01-01'                                   // обычная страница (метка = title статьи)
{ label: 'Своя метка', slug: 'guide/ch-01-01' }   // страница со своей меткой в меню
{ label: 'Группа', items: [ /* … */ ] }           // раскрывающаяся группа
{ label: 'Внешняя ссылка', link: 'https://…' }    // ссылка наружу
```

**Добавить новую статью в меню** — впиши её slug в нужное место массива:

```js
{
  label: 'Документация',
  items: [
    'guide/ch-01-01',
    'guide/ch-01-02',
    'guide/моя-новая-статья',   // ← добавили сюда (без .md, без слэшей по краям)
  ],
}
```

- **Порядок** — двигай строку выше/ниже в массиве.
- **Скрыть из меню** — просто не добавляй страницу в `sidebar.mjs` (она всё равно соберётся и будет доступна по прямой ссылке).
- **Группы** могут быть вложенными (`items` внутри `items`).

После правки `sidebar.mjs` перезапусти `npm run dev`, если он был запущен.

## 4. Ссылки

```md
[на другую статью](/guide/ch-01-02/)              # внутренняя — slug со слэшем в конце
[на якорь в статье](/guide/ch-02-05/#наборы)      # к заголовку другой страницы
[на якорь этой же статьи](#подзаголовок)          # в пределах текущей
[внешняя](https://infostart.ru/marketplace/1918555/)
```

- Внутренние ссылки — **на slug**, с `/` в начале и в конце: `/guide/ch-01-01/`.
- **Якорь** = заголовок в нижнем регистре, пробелы → дефисы, пунктуация убирается. `## Очередность подготовки` → `#очередность-подготовки`.

## 5. Картинки

Файлы клади в **`public/img/`** (можно подпапки), ссылайся по абсолютному пути `/img/…`:

```md
![Описание картинки](/img/ch_02/12_page3.png)
```

- ⚠️ **Не делай отступ ≥4 пробелов** у строки с картинкой — markdown превратит её в блок кода, и картинка не покажется.
- Картинка на отдельной строке (с пустыми строками вокруг) выводится по центру.
- Подпись — обычным абзацем под картинкой:

```md
![Схема](/img/ch_01/scheme.png)

*Рис. 1. Схема взаимодействия*
```

Схемы (`/draw_io/…`) лежат в `public/draw_io/`.

## 6. Таблицы

Обычные Markdown-таблицы (GitHub-стиль):

```md
| Поле | Тип | Описание |
|------|-----|----------|
| Имя  | Строка | Название набора |
| Признак | Булево | Доступен ли набор |
```

Выравнивание — двоеточиями в строке-разделителе: `:---` (влево), `:--:` (по центру), `---:` (вправо). В ячейках можно `**жирный**`, `` `код` ``, картинки.

---

## 7. Блоки-врезки (заметка / совет / важно / опасно)

Это **asides** Starlight. Синтаксис — `:::тип` … `:::`:

```md
:::note
Обычная заметка (по умолчанию заголовок «Заметка»).
:::

:::tip[Совет]
Полезный совет со своим заголовком.
:::

:::caution[Важно]
Предупреждение — обрати внимание.
:::

:::danger[Внимание]
Критично: так делать нельзя.
:::
```

Типы: `note` (синий), `tip` (зелёный), `caution` (жёлтый), `danger` (красный). В `[…]` — свой заголовок (опционально). Внутри можно списки, ссылки, `код`.

---

## 8. Заголовки и оглавление

```md
## Раздел           ← начинай с H2 (## )
### Подраздел
#### Под-подраздел
```

- **H1 не пиши** — заголовком страницы (H1) автоматически становится `title` из frontmatter.
- Оглавление справа строится автоматически из H2/H3. Отключить — `tableOfContents: false` во frontmatter.

---

## 9. Код

Блок кода — тройные кавычки с языком (подсветка автоматическая):

````md
```bsl
Если ЗначениеЗаполнено(Объект) Тогда
    // …
КонецЕсли;
```
````

Полезное:

````md
```bsl title="МойМодуль.bsl" {2}
А = 1;
Б = 2;            // эта строка подсветится (маркер {2})
```
````

Инлайн-код — в одинарных обратных кавычках: `` `ИмяПараметра` ``.

## 11. Расширенные компоненты (карточки, табы, шаги)

Компоненты Starlight (`<Card>`, `<CardGrid>`, `<Tabs>`, `<Steps>`, `<LinkButton>`, `<Badge>`, `<FileTree>`) работают **только в `.mdx`-файлах** (не в `.md`). Если нужны — переименуй статью в `.mdx` и импортируй:

```mdx
---
title: "Пример"
---
import { Card, CardGrid } from '@astrojs/starlight/components';

<CardGrid>
  <Card title="Быстро" icon="rocket">Текст карточки.</Card>
  <Card title="Просто" icon="approve-check">Ещё карточка.</Card>
</CardGrid>
```

Для обычных статей `.md` достаточно — asides, таблицы, картинки, код там есть.

---

## 12. Чего НЕ делать

- ❌ Не редактировать `/docs-llm/` — это сгенерированный артефакт (перезапишется).
- ❌ Не дублировать H1 в теле (его даёт `title`).
- ❌ Не отступать строки с картинками на ≥4 пробела (станут кодом).
- ❌ Не использовать `<Card>` и другие компоненты в `.md` — только в `.mdx`.
- ❌ Кириллица/пробелы в именах файлов — используй латиницу-дефис.

---

## 13. Превью и публикация

```bash
npm install          # один раз
npm run dev          # просмотр на http://localhost:4321 (горячая перезагрузка)
npm run build        # проверить, что собирается без ошибок
```

После крупных правок (много файлов) перезапусти dev: `pkill -f "astro dev"; npm run dev`.

Публикация: коммит → PR в `master` → после merge `deploy-site.yml` выкатит сайт на printwizard.ru, а `docs-llm` пересоберётся автоматически.

---

## 14. Чеклист перед коммитом

- [ ] У статьи есть `title` во frontmatter.
- [ ] Новая статья добавлена в `src/sidebar.mjs` (если нужна в меню).
- [ ] Внутренние ссылки — на slug со слэшами (`/guide/ch-01-01/`), якоря рабочие.
- [ ] Картинки в `public/img/`, путь `/img/…`, строка без отступа.
- [ ] `npm run build` проходит без ошибок.
