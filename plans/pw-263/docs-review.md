## Журнал ревизии

### 2026-05-31 — технический/рендеринг-проход по всем 70 страницам (завершён)

Системные проблемы миграции, найденные и исправленные глобально в скрипте:

1. **Картинки в code-блоке (критично).** В исходниках ВСЕ `<img>` были с
   отступом 4 пробела → после конвертации в markdown превращались в code-блоки,
   то есть **все standalone-картинки доков не рендерились**. Добавлен де-отступ
   standalone-картинок. Проверено: картинок внутри списков нет (де-отступ безопасен).
2. **Схемы draw_io.** `docs/draw_io` не копировался в `public` → схема 1.1 была
   битой. Добавлено копирование в `public/draw_io`.
3. **Callouts.** Не конвертировались типы `warning`/`new`/`highlight` и безымянные
   `{: .note }`. Добавлены (warning→danger, new/highlight→tip).
4. **Заголовки.** Дубли H1 и файлы с несколькими H1 (правило: 1→удалить,
   ≥2→понизить). Восстановлен потерянный заголовок в ch-01-04.
5. **Остаточные kramdown-IAL** `{: ... }` — вычищены.
6. **Ссылки.** Абсолютные ссылки на старый сайт → внутренние; URL-декод якорей.

Финальный аудит чист: нет битых ссылок/картинок, дублей H1, остатков Jekyll,
сырого HTML вне кода (XML-примеры корректно в code-блоках), дисбаланса asides.
67 тестов и `migrate --check` зелёные. Технический проход **по всем разделам**
выполнен (статусы ☐ в таблицах — про смысловую вычитку контента, см. ниже).

### 2026-05-31 — пустые/тонкие страницы (решено)

Причина пустых обложек: Just-the-Docs автоматически выводил список подстраниц на
`has_children`-страницах, Starlight — нет.

- **Обложки разделов** `/guide/`, `/history/`, `/preference/`, `/licensing-faq/` →
  вступление + авто-список подстраниц («В этом разделе»). `licensing-faq` оставлен
  обложкой (а не скрыт), т.к. у него реальная подстраница «Community версия».
- **Листья-заглушки** `/future/`, `/objects/` → скрыты из сайдбара (доступны по
  прямой ссылке), пока нет контента.
- `privacy`, `user-eap` — уже скрыты (`nav_exclude`), остаются «в разработке».

Итог: пустых страниц в навигации нет. 4 оставшиеся тонкие (future, objects,
privacy, user-eap) — скрыты, ждут контента от владельца.

### Осталось — смысловая вычитка контента (предметная область владельца)

- **8 тонких страниц** (контент источника, не баги): `/guide/`, `/history/`,
  `/preference/` (обложки разделов, только заголовок); заглушки «в разработке»
  `privacy`, `user-eap`, `future`, `licensing-faq`, `objects`.
- Актуальность версий, терминология, формулировки — по разделам, с владельцем.
