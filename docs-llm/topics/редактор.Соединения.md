# Соединения

Пятая страница формы. Дерево связей между наборами. Цель — расширить «Доступные поля» наборов: добавить в левый набор поля правого набора через имя соединения.

> Важно: соединения **не добавляют колонки в сам набор** (внутри pw_Исполнителя каждый набор — таблица значений). Соединения добавляются в дерево «Доступных полей» левого набора в виде вложенного «соединённого» набора с именем `ИмяСоединения`. См. «Доступные поля».

### Структура строки дерева

| Колонка (синоним) | Внутреннее имя | Назначение |
| --- | --- | --- |
| Левая сторона (набор / поле) | `ДеревоСоединений.ИмяЛево` | Набор (или поле — в зависимости от уровня дерева) слева |
| Правая сторона (набор / поле) | `ДеревоСоединений.ИмяПраво` | Набор (или поле) справа |
| Соединение | `ДеревоСоединений.ИмяСоединения` | Имя вложенного «доступного» набора. Используется в пути доступа: `НаборыДанных.ИмяЛевогоНабора.ИмяСоединения.ИмяПоляПравогоНабора` |
| № * | `ДеревоСоединений.НомерСтроки` | Номер строки в дереве (служебный) |

### Команды добавления

| Команда | Что добавляет |
| --- | --- |
| Добавить соединение → Связь наборов | Соединение целиком — аналог соединения таблиц в SQL-запросе |
| Добавить соединение → Связь полей | Условие соединения — пара полей внутри уже существующей связи наборов |

### Порядок строк в дереве

Можно менять стандартными командами `MoveUp` / `MoveDown`. **На исполнение не влияет.**

### Типы наборов в соединении

Тип источника наборов (запрос / алгоритм) и вид набора (первая строка / многострочная коллекция / …) — **не ограничивают** возможность соединения. Внутри pw_Исполнителя любой набор — таблица значений.

### Свободно стоящие наборы

Набор не обязан быть упомянут в дереве соединений. По умолчанию **все наборы — свободные**.
